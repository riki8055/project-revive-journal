
# Skills Progression: Month 1 → Month 9

This list shows **what skills are actually strengthened**, not just “topics covered”.  
Each month builds **engineering depth, failure tolerance, and decision-making ability**.

---

## Month 1: Programming & Engineering Mindset Foundations

### Technical Skills
- JavaScript fundamentals (variables, functions, async, error handling)
- Writing predictable, debuggable code
- Manual debugging using stack traces and logs
- Understanding runtime errors vs logic errors

### Engineering Skills
- Thinking in failure scenarios
- Reading error messages instead of ignoring them
- Breaking code intentionally to understand behavior
- Distinguishing *symptoms* vs *root causes*

### Professional Skills
- Writing clear commit messages
- Maintaining a learning/debug diary
- Explaining *why* code behaves the way it does

---

## Month 2: Web & System Basics

### Technical Skills
- HTTP, request/response lifecycle
- DOM manipulation without frameworks
- Client–server interaction
- Basic performance profiling (network + rendering)

### Engineering Skills
- Designing simple system boundaries
- Identifying frontend vs backend responsibility
- Understanding latency, failures, and retries
- Logging over blind console debugging

### Professional Skills
- Writing architecture explanations
- Documenting known limitations
- Communicating system flow clearly

---

## Month 3: Frontend Engineering Fundamentals (React)

### Technical Skills
- Component-based architecture

- Managing side effects safely

### Engineering Skills
- Debugging re-render issues
- Preventing infinite loops and stale state

### Professional Skills
- Code readability and maintainability
- Reasoning about UI behavior under stress
- Writing self-review notes for components
